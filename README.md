# Onboarding Demo

Static demo build for sharing the onboarding flow with test users.

## What works on GitHub Pages

- Step 1 to Step 5 UI flow
- Invite-link generation
- Form persistence in browser storage
- Resource links and checklist state rehydration
- Static demo mode without Monday credentials

## Publish to GitHub Pages

1. Create an empty GitHub repository.
2. Push this folder to the `main` branch.
3. In GitHub, open Settings > Pages.
4. Set Source to `Deploy from a branch`.
5. Choose `main` and `/ (root)`.
6. Save and wait for the Pages URL to appear.

## Local live mode

When opened on `localhost`, the app continues using the local Node server and Monday-backed routes.
