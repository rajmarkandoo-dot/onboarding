How the secure invite flow works

Front end + backend contract
1

Client opens a magic link

The link contains a random token such as ?invite=8f3Kq91Lm. No venue names are exposed in the URL.

2

Backend validates the token

The page calls an API to confirm the invite exists, is active, and belongs to the correct venue and user.

3

Progress is saved and resumed

Edits are written to storage, so the same invite can reopen the same onboarding state later.
